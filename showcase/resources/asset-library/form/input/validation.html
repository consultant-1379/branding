Input Validation

The HTML5 inputs support the validation feature. It is based on attributes such as type, pattern or required.

HTML

<input type="text" class="ebInput" placeholder="lowercase" pattern="[a-z]+"/>
<span class="ebInput-status">
  <span class="ebInput-statusError"></span>
</span>

The tag "ebInput-status" can contain specific message for the information, valid or invalid status.

email Invalid email

HTML

<input type="email" class="ebInput" placeholder="e-mail"/>
<span class="ebInput-status">
  <span class="ebInput-statusInfo">email</span>
  <span class="ebInput-statusError">Invalid email</span>
</span>
OK Invalid email

HTML

<input type="email" class="ebInput" placeholder="e-mail"/>
<span class="ebInput-status">
  <span class="ebInput-statusOk">OK</span>
  <span class="ebInput-statusError">Invalid email</span>
</span>

There are situations where you need to hide the validity icon / message, use the modifier "ebInput-status_hide".

Lowercase OK Error input: ebInput_borderColor
status: ebInput-status_hide

If you need to force one message to show, e.g. two passwords mismatch. Use the border modifier "ebInput_borderColor" / "ebInput_borderColor_red" on the input field and "ebInput-status_info" / "ebInput-status_ok" or "ebInput-status_error" on the status.

Lowercase OK Error input: ebInput_borderColor
status: ebInput-status_info
Lowercase OK Error input: ebInput_borderColor
status: ebInput-status_ok
Lowercase OK Error input: ebInput_borderColor_red
status: ebInput-status_error
